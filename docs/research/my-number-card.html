My Number Card Data Format

Japanese Individual Number Card data layout used by the reader in this repository.

Token info: EF 0006

  1. SELECT JPKI applet D392F000260100000001.
  2. SELECT EF 0006.
  3. READ BINARY length 0x14.

Individual number: EF 0001

  1. SELECT card-info-input-support applet D3921000310001010408.
  2. SELECT EF 0011.
  3. VERIFY with 4-digit PIN.
  4. SELECT EF 0001.
  5. READ BINARY length 0x11.
Byte rangeMeaning
00x10 header marker.
10x01 tag.
20x0C length.
3...1412-byte ASCII individual number.
15...16Trailing status or reserved bytes.
